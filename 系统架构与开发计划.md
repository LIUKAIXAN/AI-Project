### 1.3 Prisma Schema（核心）

```prisma
model User {
  id            String        @id @default(cuid())
  name          String
  email         String        @unique
  password      String
  role          Role          @default(DEVELOPER)
  createdAt     DateTime      @default(now())
  assignedIssues Issue[]      @relation("Assignee")
  reportedIssues Issue[]      @relation("Reporter")
  comments      Comment[]
  notifications Notification[]
}

enum Role {
  ADMIN
  PM
  DEVELOPER
  QA
  VIEWER
}

model Project {
  id          String   @id @default(cuid())
  name        String
  description String?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
  issues      Issue[]
}

model Issue {
  id          String    @id @default(cuid())
  number      Int       // 项目内自增编号
  project     Project   @relation(fields: [projectId], references: [id])
  projectId   String
  type        IssueType
  title       String
  description String?   // Markdown
  status      IssueStatus @default(NEW)
  priority    Priority  @default(MEDIUM)
  module      String?
  assignee    User?     @relation("Assignee", fields: [assigneeId], references: [id])
  assigneeId  String?
  reporter    User      @relation("Reporter", fields: [reporterId], references: [id])
  reporterId  String
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
  comments    Comment[]
  attachments Attachment[]
  labels      IssueLabel[]
}

enum IssueType { BUG FEATURE TASK IMPROVEMENT }
enum IssueStatus { NEW SUGGESTED ACCEPTED IN_PROGRESS READY_FOR_TEST TESTING DONE }
enum Priority { URGENT HIGH MEDIUM LOW }

model Comment {
  id        String   @id @default(cuid())
  content   String   // Markdown
  author    User     @relation(fields: [authorId], references: [id])
  authorId  String
  issue     Issue    @relation(fields: [issueId], references: [id])
  issueId   String
  createdAt DateTime @default(now())
}

model Attachment {
  id           String   @id @default(cuid())
  fileName     String
  fileUrl      String
  fileSize     Int
  issue        Issue    @relation(fields: [issueId], references: [id])
  issueId      String
  comment      Comment? @relation(fields: [commentId], references: [id])
  commentId    String?
  uploadedBy   User     @relation(fields: [uploadedById], references: [id])
  uploadedById String
  createdAt    DateTime @default(now())
}

model Notification {
  id        String   @id @default(cuid())
  user      User     @relation(fields: [userId], references: [id])
  userId    String
  type      String
  content   String
  issueId   String?
  read      Boolean  @default(false)
  createdAt DateTime @default(now())
}

model Label {
  id        String       @id @default(cuid())
  name      String
  color     String
  project   Project      @relation(fields: [projectId], references: [id])
  projectId String
  issues    IssueLabel[]
}

model IssueLabel {
  issue   Issue  @relation(fields: [issueId], references: [id])
  issueId String
  label   Label  @relation(fields: [labelId], references: [id])
  labelId String
  @@id([issueId, labelId])
}
```

## 二、API 设计（RESTful）

### 2.1 API 路由

| Method | Path | 功能 | 权限 |
|--------|------|------|------|
| POST | `/api/auth/register` | 注册 | Admin |
| POST | `/api/auth/login` | 登录 | Public |
| GET | `/api/projects` | 项目列表 | Auth |
| POST | `/api/projects` | 创建项目 | Admin |
| GET | `/api/projects/:id/issues` | Issue 列表 | Auth |
| POST | `/api/projects/:id/issues` | 创建 Issue | PM/Dev/QA |
| GET | `/api/issues/:id` | Issue 详情 | Auth |
| PATCH | `/api/issues/:id` | 更新 Issue | PM/Dev/QA |
| DELETE | `/api/issues/:id` | 删除 Issue | Admin |
| PATCH | `/api/issues/:id/status` | 状态流转 | PM/Dev/QA |
| POST | `/api/issues/:id/comments` | 添加评论 | PM/Dev/QA |
| POST | `/api/upload` | 上传文件 | Auth |
| GET | `/api/users` | 用户列表 | Admin |
| PATCH | `/api/users/:id` | 更新用户 | Admin |
| GET | `/api/notifications` | 通知列表 | Auth |
| PATCH | `/api/notifications/:id/read` | 标记已读 | Auth |
| GET | `/api/dashboard/stats` | Dashboard 统计 | Auth |

### 2.2 状态流转专用 API

```
PATCH /api/issues/:id/status
Body: { "action": "suggest" | "accept" | "reject" | "start" | "complete" | "test" | "pass" | "fail" }
```

| action | 状态变化 | 允许角色 |
|--------|---------|---------|
| `suggest` | New → Suggested | PM/QA |
| `accept` | Suggested → Accepted | Developer |
| `reject` | Suggested → New | Developer |
| `start` | New/Accepted → In Progress | Developer |
| `complete` | In Progress → Ready for Test | Developer |
| `test` | Ready for Test → Testing | QA |
| `pass` | Testing → Done | QA |
| `fail` | Testing → In Progress | QA |

## 四、部署方案

### 方案 A：Vercel（推荐）

```bash
# 安装 Vercel CLI
npm i -g vercel
vercel
```

- 免费额度足够 6 人团队
- 自动 HTTPS + CDN
- 缺点：SQLite 每次部署会重置（需用 Vercel KV 或外部 DB）

### 方案 B：Docker 自部署

```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY . .
RUN npm ci && npm run build
EXPOSE 3000
CMD ["npm", "start"]
```

- 完全自主可控
- 需要一台服务器（或本地电脑）
