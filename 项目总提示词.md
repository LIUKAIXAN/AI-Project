整个系统采用：

前后端分离（Frontend + Backend）

API First

架构。

具体技术选型：

优先选择 Next.js 全栈框架。

Next.js 是目前最成熟的全栈框架之一，核心优势在于：
- 一个项目同时包含前后端，不需要分开维护两个仓库。
- 前端 React + 后端 API Routes / Server Actions，开发效率高。
- 天生支持 API First 架构，Route Handler 即可编写 RESTful API。
- AI 辅助开发（Cursor、Claude Code 等）对 Next.js / React 生态支持最成熟。
- 社区庞大，文档完善，学习成本低，中小团队友好。
- 天然适合构建 Linear、GitHub、Notion 风格等现代 SaaS 界面。
- 后期可轻松扩展 AI 能力（Vercel AI SDK 原生支持）。

数据库使用 SQLite（本地轻量数据库）：
- 无需额外安装数据库服务，零运维负担。
- 非常适合中小团队内部工具，单文件即可存储所有数据。
- 后期如需切换 PostgreSQL，仅需修改连接字符串（Prisma ORM 支持无缝迁移）。
- 配合 Prisma ORM，拥有完整的类型安全和迁移管理能力。

十、技术栈推荐

基于以上原则，推荐以下技术栈：

前端/后端框架：Next.js（App Router）
UI 框架：Tailwind CSS + shadcn/ui（组件库，风格简洁，与 Linear 类似）
数据库：SQLite（本地开发）→ 后期可升级为 PostgreSQL
ORM：Prisma（类型安全，迁移管理，支持 SQLite/PostgreSQL 切换）
认证：NextAuth.js / Auth.js
部署：Vercel（免费额度足够）或 Docker 自部署

推荐理由总结：

| 技术 | 为什么选 | 适合小团队 | 易维护 | 适合 AI 开发 |
|------|---------|-----------|--------|-------------|
| Next.js | 全栈一体，生态最成熟 | ✅ | ✅ | ✅ |
| Tailwind CSS | 原子化 CSS，开发快 | ✅ | ✅ | ✅ |
| shadcn/ui | 代码级组件，完全可控 | ✅ | ✅ | ✅ |
| Prisma | 类型安全，迁移管理 | ✅ | ✅ | ✅ |
| SQLite | 零部署，零维护 | ✅ | ✅ | ✅ |
| NextAuth.js | 开箱即用认证 | ✅ | ✅ | ✅ |

不要为了追求新技术增加复杂度。

在推荐技术栈时，请说明：

为什么选择它？
是否适合中小团队？
后期是否容易维护？
是否方便AI辅助开发？
